#数据库
数据库 —> 专门用来存储数据的仓库
允许我们通过一些指令来对仓库中的数据进行快速的操作. 这些指令就是SQL语句

数据库类型:关系型数据库(结构化数据库); 对象数据库,内存数据库 

数据库特点:
1.关系型数据库是一堆表的集合
2.数据库量可能会很大, 数据每条可能会很复杂
3.相对于程序, 数据库比较独立

数据库里比较常见的概念:
1.表
数据库就是一堆表的集合,
2.字段(数据的分段)
和列相同
3.第几条数据
4.非空
5.唯一
6.主键 主键一般也是索引, 可以加快查询速度
7.外键 一个表的主键可以作为另外一张表的外键, 实现关联查询
8.默认值
9.CHECK 在数据库中, CHECK约束是指约束表中某一个或者某一些字段接收的数据值, 或者数据格式
10.自增序列

SQlite

专门为移动端设计的, 麻雀虽小, 五脏俱全;
他是一种无类型的数据库, 为了规范性, 我们建议加上类型
SQL 提供了一种近似类型
SQLite 提供了五大基本类型

亲缘规则:
1. 如果字符串中包含INT类型, 那么该字段的亲缘类型为Integer(整形就用INTEGER).
2. 如果字符串中包含CHAR CLOB 或者 TEXT,那么该字段的亲缘类型为VARCHAR(文本类型)
3. INSERT INTO table系统的关键字用大写, 小写的是名字
4. 如果字符串中包含BLOB, 那么该字段的亲缘类型为NONE(二进制文件)
5. 如果字符串中包含REAL, FLOAT DOUBLE, 那么该字段的亲缘类型为REAL(浮点数)
6. 其余情况下, 字段的类型为NUMERIC

作用也只有增删改查而已

数据库基本操作:
关注点: 简单的CRUD(增删改查) 外键关联, 约束, 级联查询(join left right), 数据库游标, 触发器, 存储过程, 数据泵还原(导出还原), 用户创建, 权限升级
(优先级 12);
创建 增删改查
创建:
CREATE TABLE student(name TEXT NOT NULL, gender TEXT);
增加
1.
INSERT INTO student(name, gender) value('陈肖东', '女');
2.
INSERT INTO student VALUE('陈肖东', '女');
删除
DELETE TABLE student;
DELETE FROM student WHERE name = '陈肖东';
改
UPDATE student SET gender = '男' WHERE name = '陈肖东';
查询
SELECT *(* 代表所有) FROM student WHERE = '女';

4. 执行SQL语句
5. 关闭数据库
